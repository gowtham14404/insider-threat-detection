// System Architecture

How It Works

Three layers of monitoring feed into a dual AI engine that scores every user against their own behavioral baseline — every 15 seconds.

01
📡
Data Collection Agent
Runs silently on endpoint. Captures keystroke timing, mouse dynamics, and process behavior. Raw keystrokes and screen coordinates are never stored — only mathematical derivatives.
pynput + psutil
02
🧠
Dual AI Scoring Engine
Two AI models work in parallel. One-Class SVM catches sudden anomalies instantly. LSTM Autoencoder catches gradual behavioral drift by analyzing sequences over time.
OCSVM + LSTM
03
📊
SOC Dashboard
Real-time threat scores for every monitored user. Color-coded alert levels, live score charts, and a scrolling alert feed. Auto-refreshes every 3 seconds.
FastAPI + HTML5
// Machine Learning

Dual AI Model Architecture

Each user gets six models trained on their behavior — two models per data modality. Both must agree before escalating an alert.

🎯
One-Class SVM
Fast Snapshot Detector
Learns the boundary of "normal" behavior using a radial basis function kernel. Scores each incoming snapshot independently. Excellent at catching sudden, obvious anomalies like a user suddenly running network scanning tools.

OneClassSVM(kernel='rbf', nu=0.08, gamma='scale')
<1ms
Latency
50

45%
Ensemble weight
🔮
LSTM Autoencoder
Deep Sequence Detector
Learns to compress and reconstruct sequences of normal behavior. High reconstruction error = the current sequence doesn't match learned patterns. Catches subtle, gradual drift that OCSVM misses.

Input→Encoder→Bottleneck(8)→Decoder→Reconstructed
~5ms

Ensemble weight
Final Score = OCSVM × 0.45 + LSTM × 0.55
Threat Score = Keystroke(30%) + Mouse(20%) + System(50%)
6 total models per user · Weighted ensemble · Per-modality scoring
Why two models?

OCSVM is fast but memoryless — it treats each moment independently. LSTM understands that behavior is a sequence. A user who always checks email then opens Excel is flagged when they suddenly open PowerShell first — even if each individual action looks normal alone.
// Biometric Features

What Gets Monitored

Every feature is a mathematical derivative — never the raw input. Your passwords and messages are completely invisible to this system.

⌨️
Keystroke Dynamics
Hold Time — how long each key is pressed (ms)
Flight Time — gap between key release and next press
Press-Press Time — rhythm between keypresses
Typing Speed — characters per minute
Key Class — ALPHA/DIGIT/SPECIAL (not the actual key)
Bigram consistency — same key-pair timing variance
🖱️
Mouse Dynamics
Velocity — cursor movement speed (px/sec)
Acceleration — rate of speed change
Jitter — high-frequency direction changes (stress indicator)
Curvature index — path efficiency metric
Click rate — clicks per second over rolling window
Double-click interval — characteristic timing signature
💻
System Behavior
Process categories — browser/dev/office/recon
Network connections — count of active connections
CPU intensity — unusually high compute usage
Risk score — weighted sum of high-risk process names
File access patterns — extension categories (not content)
USB insertion events — removable media detection

// Threat Classification

Four-Tier Alert System

Dynamic thresholds classify every user every 15 seconds. Scores are persistent — a brief spike doesn't trigger an alert, sustained anomaly does.

✅
BASELINE
0.00 — 0.35
Normal behavior detected. No action required. Continue standard monitoring.
👀
WATCH
0.35 — 0.52
Mild deviation from baseline. Could be stress or fatigue. Increase logging frequency silently.
⚠️
SUSPICIOUS
0.52 — 0.72
Notable behavioral anomaly detected. SOC analyst should review this session immediately.
🚨
CRITICAL
0.72 — 1.00
High-confidence threat detected. Escalate to security team. Consider privilege suspension.
// Privacy by Design

What We Never Store

Designed with GDPR Article 25 principles. The system monitors behavior, not content. Your private data stays private.

🔒 NEVER Captured
❌
Actual keystrokes (passwords, messages, documents)
❌
Screen coordinates (where your cursor physically was)
❌
File contents (what was inside opened files)
❌
Network packet contents (what was sent/received)
❌
Screenshots or screen recordings
❌
Browser history or URLs visited
📊 Only Mathematical Derivatives
✅
Key hold duration in milliseconds (e.g. "95ms")
✅
Mouse velocity in pixels/second (e.g. "350px/s")
✅
Process name only (e.g. "chrome.exe was running")
✅
File extension category (e.g. ".xlsx was accessed")
✅
Connection count (e.g. "12 active connections")
✅
Aggregated statistics over 15-second windows
// Applications

Real-World Use Cases

From enterprise SOC teams to academic research — behavioral biometrics has wide application in modern cybersecurity.

🏢
Enterprise SOC
Monitor hundreds of employees. Get real-time alerts when someone accesses sensitive data in unusual ways or at unusual times.
🔑
Account Takeover Detection
Even with valid credentials, an attacker types and moves differently. The system flags behavioral mismatches within minutes.
🕵️
Privileged Access Monitoring
Extra scrutiny on admin accounts. When a privileged user suddenly starts running recon tools, the AI catches it immediately.
📤
Data Exfiltration Prevention
Unusual file access patterns combined with high network activity and FTP/SCP processes triggers a CRITICAL alert automatically.
🔐
Zero Trust Architecture
Continuous behavioral authentication. Not just "who are you?" at login, but "are you still you?" every 15 seconds.
📋
Compliance & Forensics
HIPAA, SOX, PCI-DSS behavioral audit trails. Post-incident timeline reconstruction from persistent behavioral logs.
