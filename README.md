## рҹҺҜ What This Project Does

This system creates a **"Computational DNA"** profile for each user вҖ” learning *how* they use their computer, not *what* they do. It then uses two AI models to detect when someone's behavior suddenly changes, which can indicate:

- рҹ”‘ **Account takeover** вҖ” someone else using stolen credentials
- рҹҳӨ **Disgruntled employee** вҖ” erratic behavior before a data leak
- рҹ•өпёҸ **Lateral movement** вҖ” attacker moving through the network
- рҹ“Ө **Data exfiltration** вҖ” unusual file access + network activity

## рҹ§  AI Models Explained

### Model 1: One-Class SVM (Fast Scoring)
```python
OneClassSVM(kernel='rbf', nu=0.08, gamma='scale')
```
- **What it does:** Learns the boundary of "normal" behavior
- **Speed:** Scores in <1ms per sample
- **Best for:** Catching sudden, obvious anomalies
- **Limitation:** No memory вҖ” treats each snapshot independently

### Model 2: LSTM Autoencoder (Deep Scoring)
```
Input Sequence вҶ’ Encoder LSTM вҶ’ Bottleneck (8 units) вҶ’ Decoder LSTM вҶ’ Reconstructed Sequence
                                      вҶ“
                           If reconstruction error HIGH вҶ’ ANOMALY
```
- **What it does:** Learns the *sequence* of normal behavior over time
- **Speed:** Scores in ~5ms per sequence
- **Best for:** Catching subtle, gradual behavioral drift
- **Advantage:** Understands that "normal" is a sequence, not a snapshot

### Ensemble Scoring
```
Final Score = OCSVM(45%) + LSTM(55%)

Per modality:
  Keystroke  вҶ’ 30% weight
  Mouse      вҶ’ 20% weight  
  System     вҶ’ 50% weight  вҶҗ Most reliable signal

Alert Levels:
  0.00 - 0.35  =  вң… BASELINE    (normal)
  0.35 - 0.52  =  рҹ‘Җ WATCH       (log silently)
  0.52 - 0.72  =  вҡ пёҸ  SUSPICIOUS  (SOC review)
  0.72 - 1.00  =  рҹҡЁ CRITICAL    (immediate response)
```

## рҹ“Ҡ Features Captured

### вҢЁпёҸ Keystroke Dynamics
| Feature | Description | Privacy |
|---------|-------------|---------|
| Hold Time (HT) | How long each key is pressed | вң… Safe вҖ” no key content |
| Flight Time (FT) | Gap between key release and next press | вң… Safe |
| Press-Press Time (PPT) | Time between consecutive presses | вң… Safe |
| Typing Speed (CPM) | Characters per minute | вң… Safe |
| Key Class | ALPHA / DIGIT / SPECIAL (never the actual key) | вң… Safe |

### рҹ–ұпёҸ Mouse Dynamics
| Feature | Description | Privacy |
|---------|-------------|---------|
| Speed | Cursor velocity | вң… Coordinates discarded |
| Acceleration | Rate of speed change | вң… Safe |
| Jitter | High-frequency direction changes | вң… Safe |
| Click Rate | Clicks per second | вң… Safe |
| Curvature Index | Path efficiency metric | вң… Safe |

### рҹ’» System Behavior
| Feature | Risk Weight | Why It Matters |
|---------|-------------|----------------|
| Normal processes | Low | Baseline comparison |
| PowerShell / CMD | Medium | Recon indicator |
| nmap / masscan | High | Network scanning |
| FTP / SCP / rclone | High | Data exfiltration |
| mimikatz / psexec | Critical | Credential theft |

## рҹ”’ Privacy by Design

```
What NEVER gets stored:
  вқҢ Actual keystrokes (passwords, messages, documents)
  вқҢ Screen coordinates (where your cursor was)
  вқҢ File contents (what was in the files you opened)
  вқҢ Network packet contents

What DOES get stored:
  вң… Timing numbers (e.g., "key held for 95ms")
  вң… Velocity numbers (e.g., "mouse moved at 350px/s")
  вң… Process names (e.g., "chrome.exe was running")
  вң… File extensions (e.g., ".xlsx was accessed")
```

**Compliance:** Designed with GDPR Article 25 (Privacy by Design) principles. 90-day automatic data retention enforcement.
